# System Prompt Generation

This document explains how the system prompt is constructed in the Roo Code project.

## Main Generation Function

The system prompt is generated by [`SYSTEM_PROMPT()`](src/core/prompts/system.ts:107) which calls [`generatePrompt()`](src/core/prompts/system.ts:28).

## System Prompt Template Structure

```
{roleDefinition}

{markdownFormattingSection}

{toolDescriptions}

{toolUseGuidelines}

{mcpServersSection}

{capabilitiesSection}

{modesSection}

{rulesSection}

{systemInfoSection}

{objectiveSection}

{customInstructions}
```

## Variable Sources and Placeholders

### 1. Role Definition

- **Source**: [`getModeSelection(mode, promptComponent, customModes)`](src/shared/modes.ts:244)
- **Fallback**: Built-in mode's `roleDefinition` field
- **Example**: `"You are Roo, a highly skilled software engineer..."`

### 2. Markdown Formatting Section

- **Source**: [`markdownFormattingSection()`](src/core/prompts/sections/markdown-formatting.ts:1)
- **Content**: Static markdown linking rules

### 3. Tool Descriptions

- **Source**: [`getToolDescriptionsForMode(mode, cwd, supportsComputerUse, ...)`](src/core/prompts/tools/index.ts:50)
- **Variables**:
    - `{mode}` - Current mode slug
    - `{cwd}` - Current working directory
    - `{supportsComputerUse}` - Boolean for browser action availability
    - `{diffStrategy}` - Diff strategy implementation
    - `{customModes}` - Custom mode configurations
- **Logic**: Filters tools based on mode's `tools` array + `ALWAYS_AVAILABLE_TOOLS`

### 4. Tool Use Guidelines

- **Source**: [`getToolUseGuidelinesSection(codeIndexManager)`](src/core/prompts/sections/tool-use-guidelines.ts:3)
- **Variables**:
    - `{codebaseSearchAvailable}` - Boolean based on code index manager state

### 5. MCP Servers Section

- **Source**: [`getMcpServersSection(mcpHub, diffStrategy, enableMcpServerCreation)`](src/core/prompts/sections/mcp-servers.ts)
- **Condition**: Only included if mode has MCP tools (`use_mcp_tool`, `access_mcp_resource`)
- **Variables**:
    - `{mcpHub}` - MCP hub instance
    - `{enableMcpServerCreation}` - Boolean flag

### 6. Capabilities Section

- **Source**: [`getCapabilitiesSection(cwd, supportsComputerUse, mcpHub, diffStrategy, codeIndexManager)`](src/core/prompts/sections/capabilities.ts:5)
- **Variables**:
    - `{cwd}` - Current working directory path
    - `{supportsComputerUse}` - Browser action support
    - `{codebaseSearchEnabled}` - Code index availability
    - `{mcpAvailable}` - MCP hub availability

### 7. Modes Section

- **Source**: [`getModesSection(context)`](src/core/prompts/sections/modes.ts:9)
- **Variables**:
    - `{allModes}` - Array from [`getAllModesWithPrompts(context)`](src/shared/modes.ts:353)
    - Template per mode:
        ```
        * "{mode.name}" mode ({mode.slug}) - {mode.whenToUse || mode.roleDefinition.split('.')[0]}
        ```

### 8. Rules Section

- **Source**: [`getRulesSection(cwd, supportsComputerUse, diffStrategy, codeIndexManager)`](src/core/prompts/sections/rules.ts:48)
- **Variables**:
    - `{cwd.toPosix()}` - Normalized current working directory
    - `{editingTools}` - Dynamic list based on available diff/editing tools
    - `{codebaseSearchRule}` - Conditional codebase search priority rule

### 9. System Info Section

- **Source**: [`getSystemInfoSection(cwd)`](src/core/prompts/sections/system-info.ts:6)
- **Variables**:
    - `{osName()}` - Operating system name
    - `{getShell()}` - Default shell
    - `{os.homedir().toPosix()}` - Home directory path
    - `{cwd.toPosix()}` - Current workspace directory

### 10. Objective Section

- **Source**: [`getObjectiveSection(codeIndexManager, experiments)`](src/core/prompts/sections/objective.ts:4)
- **Variables**:
    - `{codebaseSearchInstruction}` - Conditional search instruction
    - `{commandInstruction}` - Conditional CLI command instruction

### 11. Custom Instructions

- **Source**: [`addCustomInstructions(baseInstructions, globalCustomInstructions, cwd, mode, options)`](src/core/prompts/sections/custom-instructions.ts:183)
- **Variables**:
    - `{language}` - User's language preference
    - `{globalCustomInstructions}` - User's global settings
    - `{modeCustomInstructions}` - Mode-specific instructions
    - `{ruleFiles}` - Content from `.roo/rules/`, `.roorules`, mode-specific rule files
    - `{rooIgnoreInstructions}` - .rooignore file instructions

## Custom System Prompt Override

If a custom system prompt file exists, the generation is simplified:

```
{roleDefinition}

{fileCustomSystemPrompt}

{customInstructions}
```

**Source**: [`loadSystemPromptFile(cwd, mode, variablesForPrompt)`](src/core/prompts/sections/custom-system-prompt.ts)

**Variables for custom prompt**:

- `{workspace}` - Current workspace directory
- `{mode}` - Current mode slug
- `{language}` - User's language preference
- `{shell}` - Default shell
- `{operatingSystem}` - OS type

## Tool Selection Logic

Tools are selected via [`getToolDescriptionsForMode()`](src/core/prompts/tools/index.ts:50):

1. **Mode tools**: From `config.tools` array (filtered by [`isToolAllowedForMode()`](src/shared/modes.ts:271))
2. **Always available**: From [`ALWAYS_AVAILABLE_TOOLS`](src/shared/tools.ts:188)
3. **Conditional exclusions**:
    - `codebase_search` removed if code index not configured
    - Tools filtered by file restrictions and experiments

## Function Parameters

The main `SYSTEM_PROMPT()` function accepts these parameters that become template variables:

```typescript
SYSTEM_PROMPT(
  context: vscode.ExtensionContext,
  cwd: string,
  supportsComputerUse: boolean,
  mcpHub?: McpHub,
  diffStrategy?: DiffStrategy,
  browserViewportSize?: string,
  mode: Mode = defaultModeSlug,
  customModePrompts?: CustomModePrompts,
  customModes?: ModeConfig[],
  globalCustomInstructions?: string,
  diffEnabled?: boolean,
  experiments?: Record<string, boolean>,
  enableMcpServerCreation?: boolean,
  language?: string,
  rooIgnoreInstructions?: string,
  partialReadsEnabled?: boolean,
  settings?: Record<string, any>
)
```

All these parameters flow through to the various section generators as template variables.
