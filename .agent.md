# Roo Code - AI Assistant Development Guide

## Project Overview

**Roo Code** is a TypeScript VSCode extension serving as an AI coding assistant with multi-LLM provider integration. Built as a Turborepo monorepo emphasizing modularity, performance, and i18n support.

**Tech Stack**: TypeScript, VSCode Extension API, React, Tailwind CSS, pnpm (v10.8.1), Node 20.19.2

## Essential Commands

### Development

```bash
# Setup
pnpm install
pnpm run build

# Development servers
pnpm run dev --filter @roo-code/web-roo-code    # Main web app
pnpm run dev --filter @roo-code/web-evals       # Evaluation system
pnpm run dev --filter @roo-code/web-docs        # Documentation

# VSCode extension
pnpm run bundle      # Bundle extension
pnpm run vsix        # Package for marketplace
pnpm run vsix:nightly # Nightly build
```

### Testing & Quality

```bash
pnpm run test                                    # Full test suite
pnpm run test --filter <package-name>           # Package-specific
pnpm run test:ci --filter @roo-code/vscode-e2e  # E2E tests
pnpm run lint                                    # Linting
pnpm run check-types                             # Type checking
pnpm run format                                  # Code formatting
```

## Architecture

### Monorepo Structure

```
src/                     # Main VSCode extension
├── api/                # LLM provider integrations
├── core/               # Core functionality (tools, prompts, webview)
├── integrations/       # VSCode API integrations
├── services/           # Business logic
├── i18n/               # Internationalization
└── utils/              # Utilities

webview-ui/             # React UI
apps/                   # Standalone applications
packages/               # Shared libraries
```

### Key Principles

- **Modularity**: Distinct modules with single responsibility
- **Type Safety**: Strict TypeScript with explicit types
- **Dependency Injection**: For testability
- **VSCode API Compliance**: Follow official guidelines

## Development Guidelines

### Code Standards

- Follow existing code structure and patterns
- Run tests before committing (`pnpm run test`)
- Use existing utility functions when possible
- Maintain backward compatibility
- Never disable lint rules without approval
- Use TypeScript strict mode
- Implement proper error handling with context logging

### Naming Conventions

- **Variables/Functions**: camelCase
- **Types/Classes**: PascalCase
- **Files**: kebab-case
- **Tests**: `.spec.ts` or `.test.ts`

### Import Organization

1. External dependencies
2. Internal modules (relative paths)
3. Types

## Testing Framework

**Primary**: Vitest with globals (no imports needed for `describe`, `test`, `it`)

- Tests in `__tests__/` or `.spec.ts`/`.test.ts` files
- VSCode API mocks in `src/__mocks__/`
- Must run from directory with vitest in `devDependencies`
- All new code requires test coverage

### Common Testing Tasks

- Bug fixes should include test updates
- New features should have corresponding tests
- UI changes should be tested in both light and dark themes
- Always consider edge cases and error handling

## Core Features

### Context Management

**Smart Context Optimization** (`src/core/sliding-window/smart-context-optimization.ts`):

- Preserves provider cache hits (up to 90% cost savings)
- Protects last 4 messages from optimization
- Removes technical failures, preserves learning feedback

### Condense System

**Advanced Context Condensing** (`src/core/condense/index.ts`):

- 5 conversation types with pattern recognition
- 4-tier information classification (CRITICAL → REMOVABLE)
- Adaptive message selection (3-10 messages based on complexity)
- 65-80% compression ratio

### Codebase Indexing

**Architecture**: Scanner → File Watcher → Cache Manager → State Manager → Orchestrator

- Thread-safe with mutex protection
- Event priority system (Delete > Change > Create)
- Smart state transitions preserving progress
- Vector store integration for embeddings

## Tool Development

### Structure

- **Tools**: `src/core/tools/[toolName]Tool.ts`
- **Prompts**: `src/core/prompts/tools/[tool-name].ts`
- **Registration**: Add to assistant message presenter
- **Types**: Update shared tool types

### Key Tools

- File operations: `readFileTool`, `writeToFileTool`, `insertContentTool`
- Code editing: `applyDiffTool`, `multiApplyDiffTool`, `searchAndReplaceTool`
- Code analysis: `codebaseSearchTool` for semantic search
- File management: `listFilesTool`, `listCodeDefinitionNamesTool`
- Task management: `attemptCompletionTool`, `newTaskTool`, `askFollowupQuestionTool`

## UI Development

### React Components

- **Framework**: React 18 with hooks
- **Styling**: Tailwind CSS utility classes only
- **Components**: Radix UI for accessibility
- **State**: React Context + hooks pattern
- **VSCode Integration**: Add CSS variables to `webview-ui/src/index.css`

### State Management Integration

**ExtensionState Updates**:

1. Add properties to `src/shared/ExtensionMessage.ts`
2. Update `packages/types/src/global-settings.ts` with Zod validation
3. Update `getState()` and `getStateToPostToWebview()` methods
4. Frontend: ExtensionStateContext → SettingsView → ContextManagementSettings

## Internationalization

**Implementation**: i18next with 18+ supported languages

- Extension strings: `src/i18n/locales/[lang]/`
- Webview strings: `webview-ui/src/i18n/locales/[lang]/`
- Package manifest: `src/package.nls.[lang].json`
- Utilities for finding missing translations

## Provider Integration

**Architecture**: Extend `BaseProvider` class

- **Location**: `src/api/providers/`
- **Transformers**: `src/api/transform/` for request/response formatting
- **Configuration**: Centralized in `src/shared/api.ts`
- Multiple providers with unified interface

### Key Provider Files

- **OpenRouter**: `src/api/providers/openrouter.ts` & `src/api/providers/fetchers/openrouter.ts`
- **Model Parsing**: Always use actual `max_completion_tokens` from API responses
- **Fallback Logic**: Use 20% of context window when max tokens not specified
